## Exploring Rust

Besides Astro, I've also been diving into **Rust**. It's a powerful language that focuses on performance and memory safety. Here is a simple example of a factorial function in Rust:

```rust
fn factorial(n: u64) -> u64 {
    (1..=n).product()
}

fn main() {
    let num = 5;
    println!("The factorial of {} is {}", num, factorial(num));
}
```

> [!success]- LC-3 Assembly Breakdown: `add.asm`
> Here is the exact line-by-line breakdown of what every single instruction is doing under the hood, and exactly *why* it has to be written that way for the LC-3 architecture.
> 
> ### The Setup
> **.ORIG x3000** : This is an "Assembler Directive." It doesn't become a CPU instruction. It simply tells the assembler, "When you convert this to binary, ensure the very first line gets loaded into memory address `x3000`." `x3000` is the standard starting address for all user-level programs in the LC-3.
> 
> ---
> 
> ### Part 1: Getting the First Input
> * **`LEA R0, PROMPT1`**: `LEA` stands for *Load Effective Address*. The assembler calculates exactly how many lines of memory away the `PROMPT1` label is. It puts that memory address directly into Register 0 (`R0`). 
>     * *Why R0?* The next command (`PUTS`) is hardcoded in the LC-3 hardware to *always* look inside `R0` to find the address of the text it needs to print.
> * **`PUTS`**: This is an alias for `TRAP x22`. The CPU looks at the memory address currently held in `R0`, goes to that address, and prints every character it finds until it hits a `x0000` (null terminator).
> * **`GETC`**: This is an alias for `TRAP x20`. It pauses the CPU's execution loop completely. When you press a key, it grabs the ASCII value of that key and shoves it into `R0`.
> * **`OUT`**: This is an alias for `TRAP x21`. Because `GETC` does *not* automatically show what you typed on the screen, we use `OUT`. It takes the ASCII character currently sitting in `R0` and prints it to the terminal.
> * **`LD R1, ASCII_NEG`**: `LD` stands for *Load*. It goes to the memory slot labeled `ASCII_NEG`, reads the actual 16-bit value sitting there (`xFFD0`), and copies it into `R1`.
> * **`ADD R2, R0, R1`**: This takes your raw keyboard input (which is sitting in `R0`) and adds the `xFFD0` (which is $-48$ in two's complement) sitting in `R1`. It stores the result in `R2`. 
>     * *Why?* If you typed '3', `GETC` put the number $51$ into `R0`. $51 - 48 = 3$. Now `R2` holds the actual mathematical integer $3$.
> * **`LD R0, NEWLINE`**: Loads the value `x000A` (the ASCII code for "Enter" or line-break) into `R0`.
> * **`OUT`**: Prints that line-break to the screen so your terminal output doesn't look like a messy, single line of text.
> 
> ---
> 
> ### Part 2: Getting the Second Input
> * **`LEA R0, PROMPT2`** & **`PUTS`**: Same as above. Loads the second prompt's address into `R0` and prints it.
> * **`GETC`** & **`OUT`**: Waits for the second key press and echoes it to the screen. The new key press overwrites whatever was previously inside `R0`.
> * **`ADD R3, R0, R1`**: Takes the new key press in `R0`, adds the $-48$ (which is thankfully still sitting in `R1` from earlier!), and stores this second integer into `R3`. Now we have our two clean numbers in `R2` and `R3`.
> * **`LD R0, NEWLINE`** & **`OUT`**: Prints another line-break.
> 
> ---
> 
> ### Part 3: The Actual Math
> * **`ADD R4, R2, R3`**: The simplest line in the program! It takes the safe, converted integer in `R2`, adds it to the safe integer in `R3`, and stores the final sum in `R4`. 
> 
> ---
> 
> ### Part 4: Printing the Result
> * **`LEA R0, RESULT_STR`** & **`PUTS`**: Loads the memory address of the "Sum: " text into `R0` and prints it to the screen.
> * **`LD R1, ASCII_POS`**: Goes to the `ASCII_POS` label, reads the value there (`x0030`, which is $+48$ in decimal), and loads it into `R1`. (This overwrites the $-48$ we were storing in `R1` earlier).
> * **`ADD R0, R4, R1`**: Takes our integer sum in `R4`, adds the $+48$ from `R1`, and puts the final answer into `R0`. 
>     * *Why R0?* Because the `OUT` command we are about to call only ever looks at `R0`. If the sum was $8$, adding $48$ makes it $56$, which is the ASCII character `'8'`.
> * **`OUT`**: Prints the character in `R0` to the screen.
> * **`LD R0, NEWLINE`** & **`OUT`**: Prints one last line-break.
> * **`HALT`**: This is an alias for `TRAP x25`. It flips the internal "running" bit of the LC-3 machine to `0`. Your Rust VM's `execute_program` loop will detect this and break out of its infinite loop.
> 
> ---
> 
> ### The Data Segment
> * **`.STRINGZ "..."`**: Another Assembler Directive. It takes your string, converts every letter into its hex ASCII equivalent, and places them into sequential memory addresses. The `Z` stands for "Zero-terminated"—it automatically adds a `x0000` to the end of the string so `PUTS` knows when to stop printing.
> * **`.FILL x...`**: This directive tells the assembler to literally just drop the 16-bit hex value directly into that specific memory slot. 
> * **`.END`**: Tells the assembler it has reached the end of the file and can stop parsing text.
